APP.A

付録A: データベーススキーマ全体図

tech-acceleration-dbを構成する13テーブルの定義、各テーブルの主要カラム、テーブル間の関係（ER図）を記す。このスキーマはGitHubリポジトリの最新版で確認できる。

tech-acceleration-dbは13のテーブルから構成されている。中核となるのは技術と科学の記録を持つ2つのコアテーブル（technologies・sciences）と、それらを結ぶ3つのリレーションテーブル（tech_to_tech・science_to_tech・quantitative_metrics）だ。さらに8つの外部データテーブルが、ノーベル賞・Ngramn頻度・技術普及・特許・R&D・論文数・Seshat政体技術・博物館アーティファクトのデータを格納する。以下に各テーブルの定義と関係を記す。

1. コアテーブル

technologies（技術マイルストーン）

カラム名型説明制約
idINTEGER主キーPRIMARY KEY AUTOINCREMENT
nameTEXT技術名称（英語）NOT NULL
name_jaTEXT技術名称（日本語）
yearINTEGER技術実証年（負数=BC）NOT NULL
year_uncertaintyINTEGER年代の不確実性（±年数）DEFAULT 0
domainTEXTドメイン（digital/communication/energy/agriculture/medical/finance/transport）NOT NULL
eraTEXT時代区分（prehistoric/ancient/medieval/early_modern/industrial/modern）NOT NULL
descriptionTEXT技術の説明（200字程度）
key_inventorsTEXT主要発明者（JSON配列）
geographic_originTEXT地理的起源（国・地域）
performance_metricTEXT定量的性能指標の名称
performance_valueREAL性能指標の値
performance_unitTEXT性能指標の単位
data_sourceTEXTデータの出典
ai_generatedINTEGERAI生成フラグ（0=検証済み、1=AI生成）DEFAULT 0
confidence_scoreREAL信頼度スコア（0.0〜1.0）DEFAULT 0.5
created_atTEXTレコード作成日時
updated_atTEXTレコード更新日時

sciences（科学的発見）

カラム名型説明制約
idINTEGER主キーPRIMARY KEY AUTOINCREMENT
nameTEXT発見・理論名称（英語）NOT NULL
name_jaTEXT発見・理論名称（日本語）
yearINTEGER発見・発表年NOT NULL
fieldTEXT科学分野（mathematics/physics/chemistry/biology/information_science/medicine/earth_science/astronomy）NOT NULL
descriptionTEXT発見の説明（200字程度）
key_scientistsTEXT主要科学者（JSON配列）
publication_refTEXT主要論文・著作の参照
data_sourceTEXTデータの出典
ai_generatedINTEGERAI生成フラグDEFAULT 0
confidence_scoreREAL信頼度スコアDEFAULT 0.5

tech_to_tech（技術間リンク）

カラム名型説明制約
idINTEGER主キーPRIMARY KEY AUTOINCREMENT
source_tech_idINTEGER起点技術IDFOREIGN KEY → technologies.id
target_tech_idINTEGER派生技術IDFOREIGN KEY → technologies.id
relation_typeTEXT関係の種類（enables/combines/improves/replaces）NOT NULL
strengthREAL関係の強度（0.0〜1.0）DEFAULT 0.5
descriptionTEXT関係の説明

science_to_tech（科学→技術リンク）

カラム名型説明制約

timelag_yearsINTEGERタイムラグ（年数）

descriptionTEXT関係の説明

quantitative_metrics（定量的成長指標）

カラム名型説明制約

tech_idINTEGER技術IDFOREIGN KEY → technologies.id
metric_nameTEXT指標名（transistors_per_chip等）NOT NULL
yearINTEGER測定年NOT NULL
valueREAL測定値NOT NULL
unitTEXT単位
sourceTEXTデータソース
growth_modelTEXTフィッティングモデル（exponential/logistic/linear）
cagrREALCAGR（年率成長率）

2. 外部データテーブル（8テーブル）

以下の8テーブルは、外部の既存データベース・研究成果から取得したデータを格納するためのテーブルだ。各テーブルはコアテーブルとの外部キー関係を持つ場合と、独立して分析可能な状態で格納される場合がある。

テーブル名件数主な内容出典
nobel_prizes 約950件 ノーベル賞受賞者・受賞分野・受賞年・受賞研究の概要。科学分野別のカウント分析に使用 Nobel Prize API (Nobel Foundation)
ngram_frequencies 約15,000件 Google BooksのNgramデータから抽出した技術・科学用語の年次出現頻度。技術の「文化的浸透」の代理指標 Google Books Ngram Viewer API
technology_adoption 約320件 主要技術の普及率時系列データ（Our World in Data）。電話・自動車・インターネット・スマートフォン等 Our World in Data
patent_counts 約8,000件 主要技術分野のIPC分類別特許出願件数の年次推移（1970〜2024年） WIPO PATENTSCOPE, USPTO
rd_indicators 約4,200件 主要国のR&D投資額・GDP比・分野別内訳（1960〜2023年） OECD Main Science and Technology Indicators
publication_counts 約12,000件 科学分野別の年間論文数・被引用数（1960〜2024年）。OpenAlexから取得 OpenAlex (https://openalex.org)
seshat_polity_tech 約2,800件 Seshat歴史データベースの政体別技術スコア（農業・冶金・建設・交通等）。紀元前3000年〜近代 Seshat Global History Databank
museum_artifacts 約6,500件 世界主要博物館の技術的アーティファクトの年代・分類データ。大英博物館・スミソニアン・国立科学博物館等 各博物館オープンデータAPI

4. 主要インデックスとパフォーマンス

本データベースは分析クエリのパフォーマンスを最適化するために以下のインデックスを設定している。technologies.domain・technologies.era・technologies.year（単体および複合）、sciences.field・sciences.year、tech_to_tech.source_tech_id・tech_to_tech.target_tech_id、science_to_tech.science_id・science_to_tech.tech_id、quantitative_metrics.tech_id・quantitative_metrics.year。

データベースファイルの現在のサイズは約45MB（圧縮前）であり、SQLiteで高速なクエリが実行可能だ。最も重いクエリ（技術ネットワークの最短経路分析）でも、通常のノートパソコンで2〜5秒で完了する。全データをPandasのDataFrameに読み込んだ場合のメモリ使用量は約200MBが目安だ。
