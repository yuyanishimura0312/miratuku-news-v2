# 経営学教科書「経営の知的地図」制作計画

## コンセプト

**タイトル**: 経営の知的地図 ── 3,458概念で読み解く経営学の全体像
**副題**: データ駆動型・概念系譜アプローチによる統合的経営学教科書

### 既存教科書との差別化

既存の経営学教科書は「分野別縦割り」構造をとる（Robbins=組織行動、Porter=戦略、Kotler=マーケティング）。入山章栄「世界標準の経営理論」は分野横断だが、個々の理論を独立に解説する百科事典型。

本書の差別化ポイント：

1. **概念間の関係を可視化する「系譜」アプローチ**: 3,458概念・5,267関係のネットワークから、概念がどう派生・対立・補完するかを読者に見せる
2. **データ駆動の構造設計**: クラスター間連結度分析に基づく章構成（恣意的でない）
3. **10クラスターの有機的統合**: 縦割りではなく、クラスター間の「橋渡し概念」を軸に経営学全体を一つの知的体系として提示
4. **232名の研究者プロファイル**: 理論と人の紐付けによる「知の人間ドラマ」
5. **実務接続**: 各章末にケーススタディと実践フレームワーク

### ターゲット読者
- MBA/経営学大学院の学生
- 経営コンサルタント・事業企画担当者
- 経営学を体系的に学び直したいビジネスパーソン
- 人類学・社会学から経営学にアプローチする研究者

## 各章の構成テンプレート

1. **導入エピソード**（500字）── 実務シーンから始まる
2. **概念の系譜**（3,000字）── 知的系譜図付きで概念の発展を追う
3. **中核理論の解説**（4,000字）── DB内の定義を引用しながら
4. **クラスター間接続**（2,000字）── 他のクラスターとの関係分析
5. **ケーススタディ**（1,500字）── 実企業の事例
6. **章末まとめと概念マップ**（1,000字）── 章内概念のネットワーク図

## 制作チーム

### Phase 1: 章構成確定・データ準備（1セッション）
- spec-writerエージェント → 各章の詳細仕様
- MG DBエージェント → 各章で使用する概念・関係リスト抽出

### Phase 2: 本文執筆（5-8セッション、4章ずつ並列）
- **執筆チーム**: 4エージェント並列（各1章担当）
  - article-team（記事制作チーム）ベースの執筆エージェント
  - 各エージェントはMG DBデータ＋章仕様書を入力とする
- **編集エージェント**: 1エージェント（品質・一貫性チェック）
- **データ可視化エージェント**: 1エージェント（系譜図・グラフ生成）

### Phase 3: 統合・校正（1-2セッション）
- 全章の統合、クロスリファレンス整備
- HTML化（textbook.htmlスタイル）
- ダッシュボードとの連携確認

### 執筆1セッションの実行パターン
```
4エージェント並列起動
  ├─ Agent A: 第N章執筆（sonnet）
  ├─ Agent B: 第N+1章執筆（sonnet）
  ├─ Agent C: 第N+2章執筆（sonnet）
  └─ Agent D: 第N+3章執筆（sonnet）
→ 完了後、統合エージェント（opus）で一貫性チェック
→ コミット・push
```

## 品質基準

- 各章10,000-12,000字（地の文、箇条書きではなく散文）
- 1章あたり30-50概念をカバー
- 各章に最低1つの概念系譜図
- 各章に最低1つのケーススタディ
- 学術的正確性（DB内の定義を根拠として引用）
- 読みやすさ（Flesch-Kincaidに相当する日本語可読性指標）
