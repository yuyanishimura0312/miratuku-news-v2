EDINET API v2 を起点に、上場企業の有価証券報告書を10年分横断的に収集・構造化したIR知識基盤。経営方針、事業等のリスク、研究開発活動、業績の概要（MD&A）といった非定量情報を、企業×年度×セクションの三次元で参照可能にすることを目的とする。

SECTION 01

概要

IR データベースは、金融庁 EDINET（電子開示システム）に集積されている上場企業の有価証券報告書を、十年というスパンで横断的に読み解くための知識基盤である。個別企業の単年度情報ではなく、業界・時代・テーマを縦断する分析を可能にすることを設計目的とする。

有価証券報告書は、上場企業が会計年度ごとに開示する最も詳細な経営情報であり、財務諸表だけでなく「経営方針」「事業等のリスク」「研究開発活動」「業績等の概要」といった構造化されたナラティブを含む。これらの記述は企業の戦略意思や業界認識を映す一次資料であり、経営学・産業分析・フォーサイト研究のいずれにおいても基盤的な参照点となる。

本データベースは、こうした記述を doc_id × edinet_code × section_tag × period_end の四つの次元で整理し、企業横断の比較・年度横断の追跡・セクション横断のテーマ抽出を可能にする。書誌メタデータ（documents）、XBRL から抽出された本文セクション（sections）、収集ログ（collection_log・extraction_log）の四層で構成される。

DATA SOURCE
EDINET API v2（金融庁・電子開示システム）から、有価証券報告書（doc_type_code=120 系）を中心に収集。XBRL 本文を構造化セクションに分解し、PDF はメタデータとともにアーカイブする。書き込みは ir-collector 側のバックグラウンドジョブが担い、本ダッシュボードは読み取り専用スナップショットを参照する。

本データベースが提供する分析軸

このデータ基盤は、ミラツクの他のデータベース群（FK・PE・PD・SI 等）と接続することで、企業の自己記述（IR）と外部観察（ニュース・政策・学術）の差分を読み解くレイヤーを提供する。とくに 研究開発活動・事業等のリスク セクションは、PESTLE 動向や政策ドメインに対する企業側の応答を追跡する一次的素材となる。

検索

本データベース上の全文検索（企業名・セクションテキスト横断）は次フェーズで公開予定である。XBRL 抽出と日本語ラベル正規化が完了次第、企業×セクション×年度のスライスインターフェースを提供する。

検索機能は Phase 6 で公開予定
sections テーブルへの全文インデックス作成、jpcrp_cor タグの日本語ラベル正規化、企業フィルタの整備が完了次第、本ページからの検索を有効化します。それまでは、SQLite 直接接続による問い合わせを推奨します。

暫定的なクエリ例

-- 研究開発活動セクションのテキスト長分布
SELECT edinet_code, filer_name, period_end, char_count
FROM sections
WHERE section_tag = 'jpcrp_cor:ResearchAndDevelopmentActivitiesTextBlock'
ORDER BY char_count DESC
LIMIT 50;
