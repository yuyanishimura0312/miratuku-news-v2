4 理論系譜と採用ポジション

生命メタファーを単一理論として扱わず、何の問いに効くかで使い分ける。Hannan-Freeman を主、Luhmann と Beer を補助、Spencer を批判対象、制度論・資源依存論を補正項として配置する。

系譜主要人物「生命」として見る対象強み採用
個体群生態学Hannan & Freeman (1977/89)組織個体群を生態学的個体群に類比多数組織の比較・存続淘汰・環境適合主理論
オートポイエーシスMaturana-Varela / Luhmann自己の要素を自己産出する閉じた過程境界・決定・意味の分析補助
サイバネティクスStafford Beer (1972-)存続可能なシステムとしての組織分権/統合・現業/方針・情報遅延の診断補助
有機体的社会観Herbert Spencer (19c)社会全体を有機体に類比長期的制度分化の枠組み批判対象
制度論・資源依存論DiMaggio-Powell / Pfeffer-Salancik正統性・依存・権力の組織「適応の物語」を正統性・模倣・強制で補正補正項

生命メタファーの 5 つの罠 (回避必須)

Reification — 組織を実体的な生命体のように扱う罠。どの作動を生命現象に類比しているのかを毎回明示する
適者生存の循環論法 — 生き残った組織を「適応していた」と呼ぶ罠。適応度・正統性・資源を結果と別に測る
個体境界の曖昧さ — 法的人格・成員・資産・通信の境界が一致しない。境界を分けて記録する
機能主義の後知恵 — 制度の存在理由を現在見える機能から逆算する罠。権威・偶然・模倣も候補に
生物学からの過剰輸入 — 淘汰・代謝・繁殖・死の対応関係を概念ごとに限定する

データベーススキーマ (PostgreSQL 16 + pgvector)

claim ベースで全主張に出典・確度・解釈者を紐づける。Stream J (Seshat の NA→Absent 撤回事件) を踏まえ value_kind を present | absent | partial | unknown | inapplicable の 5 値化。3 段階の段階リリース。

v0.1 ─ minimal claim-based core
出典トレーサブルな最小入力モデル
原初バンドから DAO まで同枠で扱う Organization と、すべての記述を polymorphic に紐づける Claim を中心。法人を標準型にせず、複数タクソノミー (Mintzberg / Hannan / Wikidata / 法人格) を併存。
organization organization_form organization_form_assignment source claim
v0.2 ─ activity / function / impact 分離
「何をしたか / 何の機能を果たしたか / 何が変わったか」
Function は 25 機能タクソノミー (Miller 20 + VSM 5) に制約。Activity は orientation (exploration / exploitation) を持つ。ImpactRecord は direction を必須化し価値判断を露見させる。
function_type activity function impact_record
v0.3 ─ relations / events / dormancy
系譜ネットワークの基盤
DiMaggio-Powell の coercive / mimetic / normative isomorphism を relation_type に物理化。Stream I を踏まえ dormancy_record で「死と継続の中間状態」を扱う。Aldrich の VSR ラベルを Event の横断軸として保持。
relation event event_organization event_relation dormancy_record
