系統5: データ・並列計算の系譜

リレーショナルDB（1970）から始まるデータ管理の系譜と、GPU並列計算の系譜は、2017年前後に「大規模分散学習」という形で合流した。データ量はMB→GB→TB→PB→EBと指数的に増大し、訓練計算量は2012年から2024年にかけて3×10⁸倍になった。この二つの成長曲線の交差点にFoundation Models（基盤モデル）が生まれ、AIエージェントの知識源となっている。

データと並列計算の合流図

データの系譜と計算の系譜は、長い間独立した軌跡を歩んでいた。データベース研究者は「いかにデータを効率よく格納・検索するか」を追求し、並列計算研究者は「いかに計算を分散・高速化するか」を追求していた。この二つの流れが、大規模機械学習という一点で合流したとき、現代のAIが誕生した。

リレーショナルDBとSQLの確立（1970–1990s）

1970年、IBMのEdgar F. Coddが発表した論文「A Relational Model of Data for Large Shared Data Banks」は、データ管理の歴史において最も影響力の大きい文書の一つである。階層型・ネットワーク型という既存のデータモデルが持つ複雑さを、集合論（タプル・リレーション）という数学的基礎で置き換えたCoddの関係モデルは、「データを正規化された表として格納し、結合操作で組み合わせる」というシンプルな原理を確立した。

1974年にIBMのSystem Rプロジェクトが開発したSEQUEL（後のSQL）は、非技術者でもデータを照会できる宣言的言語として設計された。「何を得たいか」を記述するだけで「どのように得るか」をDBMSが最適化するという発想は、抽象化の原則の見事な実践であった。1979年にOracleがSQL商用DBMSを発売し、後にMicrosoft SQL Server・PostgreSQL・MySQLが続いて、RDBMSは20世紀後半の情報システムの標準基盤となった。

1988
データウェアハウス（Bill Inmon / Ralph Kimball）
トランザクション処理（OLTP）から分析処理（OLAP）を分離する概念。スタースキーマ・列指向ストレージ・ETLパイプラインを体系化。Teradata・SASが商用実装を提供し、企業の意思決定支援システムの基盤となった。
1996–2000
Webデータと「ビッグデータ」の前夜
インターネットの急拡大により、クリックストリーム・ログ・eコマーストランザクションなど非構造化・半構造化データが爆発的に増加。RDBMSの行レベルロック・スキーマ剛性がWebスケールでは限界に達した。Google・Amazon・Yahooが独自の分散システムを開発する動機となった。

MapReduceからBig Dataエコシステムへ（2004–2014）

2003年・2004年にGoogleが発表した二つの論文——GoogleファイルシステムとMapReduce——は、大規模分散処理の設計思想を世界に公開した。「高価な信頼性の高いハードウェアではなく、安価な商用サーバを大量に並べ、故障を前提として設計する」という考え方は、当時のシステム設計の常識を覆した。MapReduceは単純な写像（Map）と集約（Reduce）という二つの関数だけで任意の分散処理を記述できることを示し、Google検索インデックスの再構築に使われていた。

Yahoo!エンジニアのDoug CuttingとMike Cafarellaがこれをオープンソースで実装したApache Hadoop（2006）は、ビッグデータ処理を商用インフラなしで実現できるようにした。Facebook・LinkedInがHadoopを採用し、「1ペタバイト以上のデータを持つ企業」が数十社規模で存在するようになった。2008年にHiveが追加されてSQL的クエリが可能になり、データエンジニアというロールが確立した。

Sparkの登場とインメモリ革命（2014）
HadoopのMapReduceはディスクへの読み書きを繰り返すため、反復的な機械学習アルゴリズムには著しく非効率だった。Berkeley AMPLabのMatei ZahariaらがRDD（Resilient Distributed Dataset）というインメモリ分散データ構造を考案し、Apache Sparkとして公開した。Hadoopと比較して単純なMapReduceで10〜100倍高速、機械学習ワークロードでは最大100倍という性能向上が実証された。SparkはMLlibによる機械学習パイプラインを内蔵し、「データ処理と機械学習を同一フレームワークで行う」という統合アプローチを示した。

GPU並列計算と大規模分散訓練の確立（2012–2020）

2012年のAlexNetが示したGPU並列計算の威力は、機械学習コミュニティをデータ管理とは独立したもう一つの軸で大きく動かした。ImageNetの120万枚の画像を処理するために必要な行列積演算は、当時のCPUでは数週間を要するが、2枚のGTX 580 GPUで5日に短縮された。この経験から、「大規模データ + 大規模並列計算 = 大規模モデル」という方程式が研究者の意識に刷り込まれた。

2017年のTransformerアーキテクチャは、並列化に最適なアーキテクチャであった。RNNやLSTMが逐次的な処理に縛られるのに対し、Self-Attentionは全ての位置を同時に処理できる。これにより数千枚のGPUを使った分散訓練が直接的な性能向上につながるようになり、「大きいモデルは小さいモデルより賢い」という「スケーリング則（Scaling Laws）」の発見へと繋がった。

Foundation Modelsとデータフライホイール（2020–現在）

2020年のGPT-3（175Bパラメータ、45TBのテキストデータで訓練）は、それ以前のモデルとは質的に異なる能力の出現を示した。指示文を与えると翻訳・要約・コード生成・質問応答を同一モデルでこなす「汎用性」は、タスクごとに個別モデルを学習する従来のアプローチの根本を覆した。この現象をStanfordの研究者たちは「Foundation Model（基盤モデル）」と命名し（2021）、単一のモデルが無数の下流タスクの基盤となる新しいパラダイムを宣言した。

現代のLLM訓練に必要なデータ量は数十TB〜百TB規模のテキストデータであり、CommonCrawl・GitHub・Books3・Wikipedia・ArXivなどから収集・清浄化される。計算量はGPT-4の訓練で推定2.15×10²⁴ FLOPと見積もられており、2012年のAlexNet（~10¹⁸ FLOP）と比較して約100万倍の増加である。この増大を支えたのが、データ管理系統（大規模データパイプライン・データセンター）と並列計算系統（GPUクラスタ・NVLink・InfiniBand）の合流であった。
